# Token Creature Concepts

> Creative ideas for token-to-creature designs

## Design Philosophy

Each token creature should:
1. **Reflect the token's identity**: Colors, symbols, or characteristics
2. **Be GameBoy-friendly**: Simple, readable at 32x32 pixels
3. **Have personality**: Cute and memorable
4. **Follow consistent style**: Limited palette, pixel art aesthetic

## Rarity System

### Visual Indicators
- **Common**: Standard colors, simple design
- **Uncommon**: One special effect (glow, pattern)
- **Rare**: Two special effects
- **Epic**: Animated sparkles, complex patterns
- **Legendary**: Full animations, multiple effects

### Shiny Variants (1/8192 chance)
- Alternate color palette
- Special sparkle effect
- Slightly different design
- "Shiny" badge in Cryptodex

## Animation Frames

### Idle Animation (2 frames)
1. Base sprite
2. Slight movement (breathing, floating)

### Encounter Animation (3 frames)
1. Fade in
2. Shake/bounce
3. Settle into idle

### Catch Animation (4 frames)
1. Surprised expression
2. Being pulled in
3. Inside pokeball (sparkles)
4. Success!

## Special Effects

### Volume-Based Effects
- **High volume**: Glow/aura effect
- **Trending**: Sparkle particles
- **New listing**: Question mark effect
- **Low volume**: Subtle appearance

### Weather Effects (Future)
- Rain: Water-type tokens more common
- Sun: Fire/gold tokens more common
- Night: Dark/mysterious tokens
- Day: Common tokens

### Dynamic Rarity Assignment
```typescript
export function getRarity(volume24h: number): string {
  if (volume24h > 1_000_000_000) return 'legendary';
  if (volume24h > 100_000_000) return 'epic';
  if (volume24h > 10_000_000) return 'rare';
  if (volume24h > 1_000_000) return 'uncommon';
  return 'common';
}
```

## AI Generation Workflow

### For MVP
1. Select top 30 tokens by volume
2. Create creature concept for each
3. Generate sprites using DALL-E 3 or Midjourney
4. Post-process to ensure:
   - Exactly 32x32 pixels
   - Limited color palette
   - Transparent background
   - Consistent style
5. Manual touch-ups in Aseprite if needed

### For Production
1. Generate sprite on first encounter
2. Cache in CDN
3. Store mapping in database
4. Fallback to placeholder if generation fails
